= 構文解析

Goに限らず、ソースコードというのは与えられた状態ではただの文字列でしかありません。
ソースコードをプログラムにとって意味のある操作可能な対象とするには、まずソースを構文解析して抽象構文木（Abstract Syntax Tree; AST）に変換し、Goのデータ構造として表現する必要があります。

いったん抽象構文木を手元に得てしまえば、任意のソースコードをプログラムから扱うのはとても簡単です。
